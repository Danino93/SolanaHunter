# 💼 ניהול ארנקים - הסבר

**תאריך:** 2026-01-25

---

## 📋 **איך זה עובד כרגע:**

### **1. ארנק יחיד (Single Wallet):**

כרגע המערכת תומכת **בארנק אחד בלבד** שהבוט עובד דרכו.

**איפה מוגדר:**
- **Backend:** `backend/.env` → `WALLET_PRIVATE_KEY`
- זה ה-private key של הארנק שהבוט משתמש בו

**איך זה עובד:**
1. הבוט טוען את ה-private key מ-`.env`
2. יוצר `WalletManager` עם הארנק הזה
3. כל הקניות/מכירות נעשות דרך הארנק הזה

---

## 🔧 **ניהול ארנקים - אפשרויות:**

### **אפשרות 1: ארנק יחיד (כרגע)** ✅

**יתרונות:**
- פשוט
- קל לתחזוקה
- מספיק לרוב המקרים

**חסרונות:**
- לא יכול להשתמש בכמה ארנקים
- צריך לשנות ב-`.env` כדי להחליף ארנק

---

### **אפשרות 2: מספר ארנקים (עתיד)** 🟢

**איך זה יעבוד:**
1. **טבלה ב-Supabase:**
   ```sql
   CREATE TABLE wallets (
     id UUID PRIMARY KEY,
     name TEXT NOT NULL,
     address TEXT NOT NULL UNIQUE,
     private_key_encrypted TEXT NOT NULL, -- מוצפן!
     is_active BOOLEAN DEFAULT false,
     created_at TIMESTAMP DEFAULT NOW()
   );
   ```

2. **UI בדף הגדרות:**
   - רשימת ארנקים
   - הוספת ארנק חדש
   - בחירת ארנק פעיל
   - מחיקת ארנק

3. **Backend:**
   - טעינת ארנק פעיל מ-Supabase
   - הצפנה/פענוח של private keys
   - החלפת ארנק דינמית

**יתרונות:**
- גמישות
- ניהול קל
- אפשרות להפריד בין ארנקים

**חסרונות:**
- מורכב יותר
- צריך הצפנה (אבטחה!)
- צריך UI

---

## ⚠️ **אבטחה - חשוב מאוד!**

### **כרגע:**
- Private key ב-`.env` (לא מוצפן!)
- רק מי שיש לו גישה לשרת יכול לראות

### **אם נוסיף מספר ארנקים:**
- **חובה להצפין** את ה-private keys!
- לא לשמור plain text במסד נתונים
- להשתמש ב-encryption (AES-256)
- Master key ב-`.env` בלבד
